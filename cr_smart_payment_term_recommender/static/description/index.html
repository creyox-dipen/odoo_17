Overview

The Smart Payment Term Recommender module helps businesses automatically suggest the most suitable payment terms for their customers during invoice creation. It analyzes historical data such as invoice payment behavior, overdue occurrences, credit note ratios, and other performance metrics to calculate a score for each customer. Based on these scores, the system matches predefined rules and recommends a payment term that aligns with the customer’s financial behavior.

This module allows users to configure flexible scoring rules for various key performance indicators (KPIs). Each KPI can have its own thresholds and score ranges, which are then linked to specific payment terms. The recommendation process is fully automated, helping finance teams reduce manual judgment, minimize payment delays, and make credit decisions more data-driven. It is ideal for companies that deal with a large number of customers and want to standardize and optimize their invoicing process.

Smart Payment Term Recommender

  • Navigate to Invoicing → Payment Matrix Configuration → Payment Term Matrix
  • User can now view all the predefined Key Performance Indicators (KPIs).
  • Here, the user can read all information related to each KPI, such as its Name, Type, Description, and more.
  • These matrices are read-only and cannot be modified.
  • Now navigate to Payment Matrix Rule.
  • Click on the New button to define rules for Payment Matrix
  • Here, the user can select a Payment Matrix from the list and assign multiple Rules to it.
  • Then, Click on Add a line, it will open popup to define rules
  • Here, the user can set the Operator, First Value (lower bound), Second Value (upper bound — only when the operator is "Between"), and a Score.
  • The user can define any number of custom rules here.
  • Click on Save Button from the left corner
  • Similarly, you can define rules for each KPI you want to include in the customer's score calculation.
  • If you select a KPI that has already been defined, a validation error will appear to prevent duplicate rule entries for the same KPI.
  • Now navigate to Payment Term Mapping.
  • Here, you can define final score ranges to map each score to a suitable payment term.
  • Click on New button.
  • It will open Payment Term Mapping window
  • Here, user can set Minimum Value, Maximum Value and Payment Term.
  • This helps determine which payment term is recommended based on the customer's overall score.
  • Then, click on Save button from the left corner
  • Define all the score ranges for selecting payment term
  • While creating a new invoice, when the user selects a customer,
  • The system automatically calculates the customer's score based on the defined payment matrix rules.
  • It then recommends a suitable payment term based on the score range configured in the payment term mapping.
  • If the customer has no past invoice records, the system will display "New Customer" and show the message "This customer has no past invoice records".
  • If the final score doesn’t match any of the defined score ranges, the system will display: "No payment term matched to this customer."
